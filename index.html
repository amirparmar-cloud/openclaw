Multi-Agent Architecture

Stop building prompts. Start building squads.

The Gateway

The central controller. It manages tool access (Browser, Files, Terminal) and provides the API bridge to communication channels like Slack and Discord.

The Workers (Souls)

Each agent has a Soul.md file defining its personality, KPIs, and boundaries. Specialized agents (Researcher, Writer, Auditor) live in isolated workspaces.

Vectorized Memory

Agents share a "Global Brain" but keep "Short-term Action Logs" private. This prevents context leakage while allowing team coordination.

The "Zero-Token" Economy

OpenClaw is model-agnostic. By integrating with **Ollama**, you can run state-of-the-art open-source models (Llama 3.2, Qwen 2.5) directly on your local hardware.

  • Cost: Zero per-token costs. Fixed electricity cost only.
  • Privacy: Data never leaves your network. Perfect for legal/financial.
  • Hybrid Routing: Use local Llama for 90% of tasks; escalate to Claude 4.6 API for high-stakes logic.

Hardware Choice: The Mac Mini M4

For most users in 2026, the Mac Mini is the perfect "Always-On" server for OpenClaw.

Why Mac Mini?

  • • Unified Memory: Fast GPU-RAM for LLMs.
  • • Power Efficiency: Runs 24/7 for pennies.
  • • Form Factor: Tucks away in a drawer.
  • • Headless Mode: Controlled via SSH/OpenClaw UI.
